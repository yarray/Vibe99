Layout switching now opens a new window instead of replacing the current session. The new window loads with `?layout=<id>` in the URL and restores the target layout on startup, while the original window remains unchanged. This prevents loss of unsaved terminal output and running processes when switching layouts. Also includes the VIB-84 fix: `layout_save` now persists `activeLayoutId` immediately, and `applyLayout` saves settings synchronously to avoid races.
