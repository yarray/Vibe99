Replace CSS animation + `will-change: opacity` with JS requestAnimationFrame for breathing light. CSS `animation` on `::after` pseudo-elements promotes each to a separate compositor layer in WebKitGTK; multiple alerted panes saturate integrated GPU memory bandwidth, causing sporadic 100% GPU spikes. `animation-play-state: paused` does not release compositor layers. The RAF approach stops completely when the window is blurred/minimized/hidden (via both `visibilitychange` and `blur` events), uses no `will-change`, and animates a dedicated `.alert-overlay` child element instead of sharing `::after` with the pane background mask.
