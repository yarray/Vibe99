Added a data-driven Settings schema validation system (`src/domain/settings-schema.ts`). Settings are now automatically validated and sanitized on load, with invalid values auto-repaired to valid defaults. Replaced manual `Number.isFinite()`/`Math.max/min` checks with schema-based validation for fontSize, fontFamily, paneOpacity, paneMaskOpacity, paneWidth, webglEnabled, breathingIntensity, and activityAlertDebounceMs. Added `ValidationReporter` interface with `ConsoleValidationReporter` and `SilentValidationReporter` for user notification control.
