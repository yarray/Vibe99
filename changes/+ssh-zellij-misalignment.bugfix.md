Fix display misalignment when restarting PTY sessions after TUI program exit. When TUI programs (Vim, Codex, etc.) exit, their escape sequences may leave xterm.js in an inconsistent state with incorrect cursor position, scrolling region, or other terminal modes. This caused display misalignment for the subsequent PTY session. The fix calls terminal.reset() before creating a new PTY, which resets all terminal state without clearing the screen content. Also improves PTY size synchronization by waiting for layout to stabilize before creating the PTY.
