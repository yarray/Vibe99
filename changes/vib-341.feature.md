[VIB-341] feat: Settings panel supports layout-level UI override writing with pin indicators

- Settings inputs now write to `layout.uiOverrides` when the current layout has an override for that field, ensuring changes are immediately visible in the current window.
- When no layout override exists, settings continue to write to global settings as before.
- Added pin icons next to overridden settings in the Settings panel to indicate which values are currently overridden by the active layout.
- Runtime consumers (`terminal-session.ts`, `pane-renderer.ts`) now use `settingsManager.resolvedSettings` so font size, font family, and pane width correctly respect layout overrides.
- Settings panel auto-refreshes when switching layouts to show the resolved values for the new layout.
